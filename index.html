Empeirias Docs Installation

Node Installation

Step-by-step guide to install the Empeirias node software and dependencies.

1. System Preparation

Update system and install basic dependencies:

sudo apt update && sudo apt upgrade -y
sudo apt install curl tar wget clang pkg-config libssl-dev jq build-essential \
bsdmainutils git make ncdu gcc chrony liblz4-tool -y

2. Install Go 1.21.6

ver="1.21.6"
wget "https://golang.org/dl/go$ver.linux-amd64.tar.gz"
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf "go$ver.linux-amd64.tar.gz"
rm "go$ver.linux-amd64.tar.gz"
echo "export PATH=$PATH:/usr/local/go/bin:$HOME/go/bin" >> $HOME/.bash_profile
source $HOME/.bash_profile
go version

3. Install emped Binary

mkdir -p $HOME/go/bin
curl -LO https://github.com/empe-io/empe-chain-releases/raw/master/v0.3.0/emped_v0.3.0_linux_amd64.tar.gz
tar -xvf emped_v0.3.0_linux_amd64.tar.gz
rm emped_v0.3.0_linux_amd64.tar.gz
chmod +x emped
mv emped $HOME/go/bin/
Tip: Verify the binary checksum matches the official release before proceeding.
Empeirias Docs Node Initialization

Node Initialization

Initialize your node and download the necessary configuration files.

4. Initialize Node

emped init YOUR_NODE_NAME --chain-id empe-testnet-2

5. Download Genesis File

wget -O $HOME/.empe-chain/config/genesis.json "https://raw.githubusercontent.com/empe-io/empe-chains/refs/heads/master/testnet-2/genesis.json"

6. Download Addrbook

wget -O $HOME/.empe-chain/config/addrbook.json "https://raw.githubusercontent.com/111STAVR111/props/main/Empeiria/addrbook.json"

Empeirias Docs Snapshot

Snapshot Restoration

Quickly sync your node using a snapshot instead of syncing from genesis.

9. Download Snapshot

sudo systemctl stop emped
cp $HOME/.empe-chain/data/priv_validator_state.json $HOME/.empe-chain/priv_validator_state.json.backup
rm -rf $HOME/.empe-chain/data
curl https://server-5.itrocket.net/testnet/empeiria/empeiria_2025-04-04_4319626_snap.tar.lz4 | lz4 -dc - | tar -xf - -C $HOME/.empe-chain
mv $HOME/.empe-chain/priv_validator_state.json.backup $HOME/.empe-chain/data/priv_validator_state.json
sudo systemctl restart emped && sudo journalctl -u emped -f -o cat

10. Check Sync Status

emped status 2>&1 | jq

If status shows false, proceed to the next steps.

Note: Snapshot restoration typically takes 30-60 minutes depending on your connection speed and hardware.
Empeirias Docs Create Validator

Become a Validator

Create your validator after your node is fully synced.

11. Create Validator

emped tx staking create-validator \
--amount 1000000uempe \
--commission-rate 0.1 \
--commission-max-rate 0.5 \
--commission-max-change-rate 0.2 \
--min-self-delegation 1 \
--pubkey $(emped tendermint show-validator) \
--moniker "YOUR_VALIDATOR_NAME" \
--chain-id empe-testnet-2 \
--from YOUR_WALLET_NAME -y
Important: Ensure your node is fully synced before creating a validator. You need at least 1,000,000 uempe to create a validator.

Empeirias Docs Cheatsheet

Empeirias Validator Cheatsheet

Essential commands for managing your Empeirias validator node.

Wallet Operations

# Create new wallet
emped keys add wallet_name

# Recover wallet from mnemonic
emped keys add wallet_name --recover

# List all wallets
emped keys list

# Check wallet balance
emped q bank balances $(emped keys show wallet_name -a)

Node Management

# Start node service
sudo systemctl start emped

# Stop node service
sudo systemctl stop emped

# Restart node service
sudo systemctl restart emped

# Check node status
sudo systemctl status emped

# View node logs
journalctl -fu emped -o cat

Validator Operations

# Check validator info
emped query staking validator $(emped keys show wallet_name --bech val -a)

# Edit validator details
emped tx staking edit-validator \
  --moniker "NEW_NAME" \
  --identity "KEYBASE_ID" \
  --details "Description" \
  --website "https://website.com" \
  --from wallet_name \
  --chain-id empe-testnet-2

# Unjail validator
emped tx slashing unjail --from wallet_name --chain-id empe-testnet-2

# Delegate more stake
emped tx staking delegate \
  $(emped keys show wallet_name --bech val -a) \
  1000000uempe \
  --from wallet_name \
  --chain-id empe-testnet-2

Transactions

# Send tokens
emped tx bank send wallet_name recipient_address 1000000uempe --chain-id empe-testnet-2

# Vote on proposal
emped tx gov vote 1 yes --from wallet_name --chain-id empe-testnet-2

# Submit proposal
emped tx gov submit-proposal \
  --title "Proposal Title" \
  --description "Proposal Description" \
  --type "Text" \
  --deposit 1000000uempe \
  --from wallet_name \
  --chain-id empe-testnet-2

Network Info

# Check sync status
emped status 2>&1 | jq

# Check node info
emped status

# Check validator set
emped q tendermint-validator-set

# Check latest block
emped q block

Empeirias Docs Troubleshooting

Troubleshooting Guide

Common issues and solutions for Empeirias nodes.

Node Not Syncing

If your node is not syncing, try these steps:

  1. Restart the node service: sudo systemctl restart emped
  2. Check your peers in addrbook.json
  3. Ensure your system time is synchronized: sudo timedatectl set-ntp on

Validator Not Active

If your validator is not active in the set:

  • Check your self-delegation amount
  • Ensure your node is not jailed
  • Verify your commission rate is within limits
Emergency: If your validator is jailed, you'll need to unjail it using emped tx slashing unjail --from YOUR_WALLET --chain-id empe-testnet-2
